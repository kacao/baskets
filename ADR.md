## Stack, build & data (ADR-001/002/021/025/050)

- **SvelteKit monolith** (`adapter-node`), no client data layer: mutations are **form actions**, pages use server `load` â€” progressive enhancement + validation co-located with mutations.
- **Multi-dialect DB via Drizzle (ADR-050)**: **SQLite (default, better-sqlite3) + Postgres (postgres-js)**, chosen by `DB_DIALECT` (`DATABASE_URL` = file path or `postgres://â€¦`). **Per-dialect schema files** â€” `schema.sqlite.ts` (canonical) + `schema.pg.ts` (byte-identical table/column _names_, only types differ) behind a `schema.ts` **facade** that casts PG onto the sqlite types so all importers keep one shape. Schema applied via `drizzle-kit push` â€” **no migration files**. **Rule: edit both schema files in lockstep.** MySQL deferred (needs a `.returning()`/upsert shim + varchar PKs).
- **Prod runs a custom `server.js`** (adapter-node handler + realtime `/ws` upgrade) as one Node process; `npm start` runs it, not `build/index.js`.
- **`withTransaction` dialect-safe atomicity primitive (ADR-057)**: better-sqlite3's `db.transaction(fn)` needs a **synchronous** callback returning `T` directly, incompatible with the async service layer; postgres-js's `transaction` is properly async. `withTransaction` picks per `DIALECT` â€” pg uses its native async `transaction`; sqlite issues a manual `BEGIN IMMEDIATE`/`COMMIT`/`ROLLBACK` on the shared connection (fine for low-concurrency use â€” not safe for overlapping concurrent transactions on one connection; **multi-tenant PRODUCTION deploys should set `DB_DIALECT=postgres`**, ADR-062). **Rule: only DB writes go inside the callback â€” fire-and-forget side effects (`dispatchEvent`, `broadcastProjectChange`, `notifyMentions`, `logActivity`, `createNotification`) run in the caller after it resolves.** So far only `updateTaskService` (custom-field write + task update + completing-cascade) is wrapped; `bulkUpdateTasks`/`moveTaskService`/`setTaskStatusService`/`createTaskService` remain sequential (follow-ons).

## Auth, API & access control (ADR-003/005/006/013/017/019/036/039/048/049)

- **BetterAuth** (TOTP + admin plugins, Drizzle adapter, auth tables in app schema); `hooks.server.ts` populates `locals.user`/`session`. **Explicit policy (ADR-048)**: 12-char password floor, 14-day sessions, prod-only rate limit; seed passwords from `SEED_*` env vars.
- **REST API** with **dual auth** (session cookie OR `Bearer bsk_â€¦` â€” 256-bit, SHA-256 hash, plaintext shown once).
- **Multi-tenant via organizations (ADR-062)**; **workspaces remain the sharing primitive INSIDE an org** â€” every workspace belongs to one organization, every project to one workspace (owner); statuses + labels stay workspace-scoped. Empty-only workspace deletion; a 0-workspace org is legal (required for org deletion).
- **Visibility = access (ADR-019, org-scoped by ADR-062)**: org membership is a prerequisite for ANY access to that org's data; org owners/admins see all of their org; plain members see workspaces they own/are granted + directly-granted projects (membership alone confers nothing). **Instance admins (`user.role='admin'`) have NO implicit tenant-data reach.** **Inaccessible reads â†’ 404** (no existence oracle), not 403 â€” including missing-project sub-resource GETs. Guard reads with `canAccessProject`; filter lists by accessible ids. No read-only tier (`canAccessProject â‰¡ canEditProject`).
- **Edit rights**: task editing needs project **access**; **structure edits** (workspace/project/view/status/milestone/label/location meta) need org owner/admin / workspace owner / a `permission` grant. Cascade: org â‡’ workspaces â‡’ projects â‡’ views; view grants don't confer visibility. Grant writes require the grantee to be an org member and stamp `permission.organizationId` (grant org == resource org).
- **One service layer, thin adapters (ADR-049)**: each domain has a service module (`src/lib/server/{tasks,statuses,labels,milestones,views,customFields,comments}.ts`) owning all validation, permission guards, writes, and side-effects (broadcast, notifyMentions, activity, recurrence, cascade), returning a discriminated `ServiceResult<T>`. Form actions and REST endpoints are **thin adapters** differing only in input parsing (FormData vs JSON) + response shaping. Full UIâ†’REST parity (ADR-036); secrets (key hash, `storagePath`) never returned; Integrations + auth/2FA excluded. **Rule: a new mutation gets a service fn + adapters â€” never a second copy of the logic.** Keep `static/llms.txt` current.
- **Export/import (ADR-039)**: JSON/CSV export (status by name); import is web-only, creates a NEW project with id-remapping, **lossy by design**. Superseded the removed **saved filters** (per-view `view.config.filters` persists instead).

## Multi-tenancy: organizations (ADR-062)

- **Tenant = organization**, one level above workspaces (`organization â†’ workspace â†’ project â†’ task`), via the **BetterAuth `organization` plugin** (tables `organization`/`member`/`invitation` + `session.activeOrganizationId`, hand-written into both dialect schemas; teams + dynamicAccessControl deliberately disabled). Users are global (one account/email) with â‰¥0 memberships; org roles **owner/admin/member live ONLY on `member.role`** â€” never `user.role`, because the admin plugin's `adminRoles` reads `user.role` and would grant instance-wide powers (list/ban/impersonate).
- **Isolation is row-level in code** (shared schema/DB): almost every table chains to the org via `project â†’ workspace.organizationId`; direct `organizationId` columns exist only where the chain breaks â€” `integration` (`unique(organizationId, type)` replaced the per-install `unique(type)`; org required in code since sqlite composite uniques ignore NULLs), `notification` (required at create; the bell filters to current member orgs), `permission` (denormalized for single-delete purges + the grant-org==resource-org invariant). Membership is checked FIRST in every guard, so a departed member's grants are **inert by construction** (purges via `afterRemoveMember`/re-join are hygiene, not the boundary).
- **Instance admin = operator, not super-tenant**: `isAdmin` split into `isInstanceAdmin` (only `/admin` + `/settings/statuses`, incl. instance-global built-in status icons) and org roles. The `'all'` sentinel was REMOVED from `accessibleWorkspaceIds`' return type so the compiler enumerated every consumer during the migration. **Documented operator powers that DO reach tenant data** (excluded from the isolation guarantee; DB-equivalent for a self-hoster): admin-plugin impersonation and the `/admin` direct member-add (server-only `auth.api.addMember`). **Roster boundary = the org** (any member can list the org roster â€” the plugin's `listMembers` is live HTTP anyway); per-project rosters (`projectAccessUserIds`) stay narrower for pickers and no longer include instance admins.
- **Active-org transport**: an **`org` cookie** mirroring the `workspace` cookie (validated against memberships, silent fallback, no oracle; `session.activeOrganizationId` exists for the plugin but the app ignores it). Org switch clears the workspace cookie; **project loads auto-align the org cookie** to an accessible project's org, which makes bell clicks/deep links/mention chips coherent with one rule. Bearer/REST requests carry no session: resource endpoints derive the org from the addressed resource; collections default to the **union of per-org access-filtered sets** (optional `?org=` filter). API keys stay per-user/org-agnostic â€” guards re-check membership per request.
- **Invitations without SMTP (the link IS the capability)**: `sendInvitationEmail` omitted; org admins copy `/invite/<id>` (UUID-entropy) from org settings. Deliberately **NO by-email invite discovery** â€” with unverified open registration, session email is attacker-chosen (the plugin's own `listUserInvitations` 403s unverified emails for the same reason); the accept-time email match is a speed bump, not authentication. Signup stays open; **0-org users are redirected to `/onboarding`** ((app)-layout guard) to create their org (plugin hooks: server-generated slug; idempotent non-throwing workspace bootstrap).
- **Migration**: one-time, **marker-gated** (`organization.metadata {"migrated":true}` written LAST), crash-resumable (idempotent steps, deterministic member ids, one `withTransaction`), runs only when legacy data exists â€” **fresh installs create nothing** (first user owns their org via onboarding). Legacy installs: all workspaces/integrations/grants/notifications stamped to `org-default`; every user materialized as a member (oldest admin â†’ owner, no-admin fallback â†’ oldest user â€” admins held no grant rows, so memberships HAD to precede removing the `isAdmin` bypass). The old `ensureDefaultWorkspace` orphan-adoption now lives ONLY inside the migration; post-marker, a null `workspaceId`/`organizationId` is a logged integrity error, never adopted.
- **Org lifecycle**: deletion is app-guarded (plugin deletion disabled) â€” owner-only, 0 workspaces, explicit transactional sweep (ALTER-added org columns have no real FK on sqlite; only member/invitation cascade). Last-owner can't leave/demote; only owners move ownership. All `/api/auth/organization/*` plugin endpoints are live surface â€” consistency runs through `organizationHooks`, not just our UI paths.

## Domain model (ADR-010/011/014/015/024/029/031/037/043/045)

- **Tasks**: one nesting level (depth 1, enforced). Parent â†’ `done` cascades to subs; deleting a parent deletes subs.
- **Statuses**: table-driven, behavior keys off `status.category` (`backlog|planned|in-progress|completed|canceled`) **never the name**. Three scopes: five fixed app-wide built-ins (name/category fixed, icon editable), workspace-scoped, project-scoped; `project_status` whitelists per project. Optional color/icon/description. **Status editor (ADR-029)** is category-grouped, drag-reorder within a category; built-ins + inherited render read-only.
- **Milestones / labels / dependencies** are plain join tables; all dependencies are **informational** ("blocked by"), never gating status. A label is EITHER workspace-scoped (shared pool, opted onto projects) OR project-scoped; optional color + icon.
- **Locations**: project-scoped, coords optional; legacy `task.location` text kept as a map fallback.
- **Custom fields (ADR-031)**: project-scoped typed fields on **tasks OR the project**; types incl. select/person/files/task/place/**rollup**. **One TEXT value per `(task,field)`** (scalar or JSON id-array for multi-capable types) over typed columns/row-per-value â€” preserves order, one-query load. Config is schemaless JSON, name unique per (project,entity), **type immutable after create**, **`appliesTo`** gates task levels. **v1 display-only** (no sort/group/filter by them); formula/button deferred via type+config. All writes funnel through one validated path (no cross-project refs).
- **Number rollup-to-parent (ADR-043)**: a `number` field can show a **parent's value as the aggregate of its sub-tasks'** values for the same field â€” **display-only, never stored**; aggregations skip a parent's stale stored value to avoid double-counting.
- **Collapsible sections (ADR-045)**: multi-value field pills, plus the **Sub-tasks** (`__subtasks`) and **Description** (`__description`) sections, collapse via one localStorage mechanism keyed **per container** (task id / project id). Multi fields + Sub-tasks default **collapsed** (Sub-tasks auto-expands on first add, ADR-056); Description defaults **expanded**. Pure client UI state.
- **Files (ADR-031)**: uploads on **local disk** (gitignored), `storagePath` never exposed. Served only via an access-gated streaming endpoint with `nosniff` + CSP `sandbox`, **server-derived MIME**, and an exec/markup denylist â€” prevents stored-XSS from our origin.
- **Views**: per-project `view` rows with **schemaless `config` JSON**; multiple views per type. Map (Leaflet) and **Flow (ADR-037, Svelte Flow)** are **client-only** (dynamic import in `onMount`) so the libs never run at SSR; Flow uses a hand-rolled longest-path layout (no dagre dep). Display-only.
- **Start/due dates** on task/milestone/project â€” optional, informational.

## Project page & editing UX (ADR-012/020/023/024/027/030/041/044/046/054/055/056)

- **Project header portaled into the shell topbar (ADR-044)** via a shared `portal` action â€” a portaled node keeps its Svelte identity + scoped styles, only its DOM parent moves. **Hidden until portaled** (the action stamps `data-portaled`; `.proj-topbar` is `display:none` until then) so it never flashes in its SSR position on hydration (ADR-055).
- **Project header** = title/icon/pin + a "â€¦" editor menu (Create / Status / Icon / Labels / Milestones / Delete). Resizable, persisted width; full-width overlay <900px (full-screen sheet above the topbar on phones, ADR-053). **Exactly one pane open at a time** (registry).
- **Task pane**: full-width Title; a **pills row** (each a reusable `Popover`; `patchTask` updates only fields present â€” no clobber); a **collapsible Description** (borderless, document-like editor); a **collapsible Sub-tasks** section (default collapsed, auto-expands on first add, header "+"). Date pills open a **custom `DatePicker` calendar** (no native `<input type=date>`); number custom fields have âˆ’/+ steppers + a currency affix + right-aligned tabular figures (ADR-056).
- **URL-addressable panes (ADR-054/055)**: the open pane is reflected in the URL (`?task=` in the 6 views, `?pane=â€¦` on the project page) via **SvelteKit shallow routing** through `src/lib/paneUrl.ts` (no `load()` re-run). **History:** opening a pane (param absentâ†’present) **pushes** one entry so Back closes the pane and restores the pre-open URL incl. the active `?view=` tab (an unconditional replace overwrote the tab's entry â€” Back jumped to the previously-viewed tab); in-pane nav + close **replace**, so drilling doesn't fill history. **Rules:** write with `setPaneUrl` (based on `window.location`, since shallow routing leaves `page.url` stale); **read with `readPaneParam`** (returns `window.location`'s value but touches `page.url` so back/forward still re-trigger) â€” reading `page.url.searchParams` directly lets an `invalidateAll()` re-run the read-effect with the stale URL and close the pane; read any URLâ†”state sentinel via `untrack`.
- **Escape & effect hygiene (ADR-055)**: modals/popovers that can open over the pane stop Escape in the **capture phase** (so it dismisses them, not the pane behind); `ConfirmModal` scopes Enter-confirm to the dialog (no danger-delete from a background field). Effects seeding local state from `page.data`-derived props (e.g. `CustomFieldValue`'s one-way-bound editor) **guard on identity/value** so `invalidateAll` churn can't reset an in-progress edit.
- **Overview & Files pages (ADR-046)**: two per-project routes with **independent loads gated by `canAccessProject`â†’404** (not the settings `canEditProject`/403) so non-editors get read-only. Overview = borderless document-like title+description (reuses the settings `updateProject`, patch-style for dates). Files = aggregates every project file (task attachments + cf uploads + project-level) via `GET`/`POST /api/projects/{id}/files` (project file = `taskId`+`fieldId` null; same validation as `POST /api/files`).
- **Table** keeps column widths (content pane scrolls instead of squishing); **resizable columns** persist to `config.colWidths` (â†’ `table-layout: fixed`). **Group by** shared across Table/List; Board adds swimlanes. TableView uses **one `<table>`** so columns align across all groups (ADR-030). Per-view **Customize** pane holds status-display / hide-empty-groups / columns (ADR-041).
- **No decorative glyphs** before entity names (a `â›“ N` blocker _count_ is fine).

## Realtime collaboration (ADR-025)

Native **WebSockets** on `/ws`; transport is **plain-ESM** (no `$lib`/TS) so it's shared by the Vite dev plugin + prod `server.js` and imports no app modules. **Sync = invalidate + refetch** (not OT): a mutation fire-and-forget broadcasts a project-changed ping, clients debounce `invalidateAll()` â€” reuses every `load()` permission filter, no delta protocol; concurrent edits are last-write-wins. Upgrade/subscribe auth by forwarding cookies to the app's own `/api/me` + `/api/projects/{id}`. Presence avatars via heartbeat.

## Cross-cutting (ADR-007/008/016/018/022/027/028/034/038/040/042/052/053)

- **Theming**: Tailwind 4 + DaisyUI 5 (light/dark); a **token bridge** maps legacy `--color-*`/`--sp-*` vars onto DaisyUI tokens so scoped `<style>` blocks follow the theme. Theme via `theme` cookie at SSR. No focus rings.
- **High contrast (ADR-038)**: a **separate axis** from light/dark (`contrast` cookie) overriding tokens to black/white + squaring large surfaces (scoped CSS, not token-zeroing).
- **Tooltips (ADR-034)**: a `use:tooltip` action over **one shared singleton node**; strips the host's native `title` to avoid an OS double.
- **ColorPicker (ADR-040)**: structural twin of IconPicker â€” controlled `{value,onSelect,onRemove}`, caller owns persistence; emits 6-digit `#rrggbb`. Replaces raw `<input type=color>`.
- **Icons (ADR-052, was iconoir/ADR-028)**: **Heroicons** (24px outline) via a generated **external SVG sprite** (`scripts/build-heroicons-sprite.mjs`) that also emits **alias symbols** for the ~60 legacy iconoir tokens still hardcoded, so existing `<Icon name>` call sites render Heroicons unchanged. Stored entity icons keep the `iconoir:` prefix as a namespace (a picked Heroicon is stored `iconoir:<heroicon-name>` â€” no data migration). **Rule: new icons use Heroicon names; reuse a legacy token only by adding it to `ALIASES`.**
- **Motion**: no animation library â€” `svelte/transition` + CSS, â‰¤200ms; `prefers-reduced-motion` kills all motion.
- **Toasts / Confirm modal (ADR-027)**: tiny app-wide primitives (`toast()`, `confirmDialog()â†’Promise<boolean>`) mounted once in the shell; the modal replaces native `confirm()`. Destructive confirms use enhance's `cancel()` (never `onsubmit`+`preventDefault`).
- **i18n**: hand-rolled, English strings **are** the keys (passthrough); `$t` + `registerDictionary` kept so a locale can be re-added. DB content never translated.
- **Integrations**: `dispatchEvent` fire-and-forget (never throws) after mutations; one row per type. Slack = incoming webhook (URL allow-listed, stored server-side, never re-exposed).
- **Touch / mobile (ADR-042/053)**: native HTML5 DnD + `oncontextmenu` are mouse-only â†’ two in-house **pointer-event** primitives replace them (`use:sortable` â€” touch engages after a still hold so swipes still scroll; `use:longpress` â€” fires the same handler as `oncontextmenu`). **Rule: don't reintroduce `draggable` for new reorder UI.** Mobile pass (ADR-053): `viewport-fit=cover` + `env(safe-area-inset-*)` on fixed/edge UI, 16px inputs to kill iOS zoom-on-focus, full-screen task sheet above the topbar, hover flyouts become click accordions on touch. **Rule: any new hover-revealed control on a tap target needs the `@media (hover:none)` neutralization** (the iOS "2-tap" bug).
- **`static/llms.txt`** documents the REST API for agents â€” keep current on any REST/status/view change.

## "@" inline-token references (ADR-047/051)

- **Storage = inline tokens, no schema change**: `@[label](kind:id)` lives in the existing plain-text columns (`task.description`/`comment.body`/`project.description`); `label` is denormalized as a delete fallback. Kinds: task/location/file/project/person, plus **`field` (ADR-051)** â€” references one of the OWNING task's own custom fields and renders a display-only `[name | live value]` chip (resolved at render time; never notifies). Parsing in `src/lib/mentions.ts` (client+server safe, tolerant).
- **Rendering = `RichText.svelte`** â€” tokenizes to text spans + real chips (**no `{@html}`**; labels interpolated, no stored-XSS; dangling ids degrade to a muted label). Also **auto-links bare URLs** via `linkify()` (http(s)/`www.`/email â†’ real `<a target=_blank rel=noopener>`, hrefs restricted to http(s)/mailto). Editing = **`MentionEditor.svelte`**, a contenteditable that shows inline pills while typing, serializes DOMâ†’token on input, and anchors the caret via the native Selection API.
- **Server (`notifyMentions`)**: notifies only **newly-mentioned** `person:` refs, not the actor, and **only project-accessible users** (`person:<id>` is attacker-controlled text â€” a hand-typed token can't notify/leak to arbitrary users). Hooked into `patchTask`, comment POST/PATCH, and `updateProject`.

## Tooling, tests & CI (ADR-058/059/060/061)

- **CI + integration gating (ADR-058)**: GitHub Actions (`.github/workflows/ci.yml`) runs `check` + `test:unit` on push/PR to `dev`/`main`. Server-dependent suites (`test:integration`/`test:e2e`) are gated behind **`RUN_INTEGRATION=1`** so an absent/unseeded server FAILS loudly instead of skipping-as-passing (the old silent skip hid regressions). `npm run verify` (check + unit) is the fast local mirror. **Follow-up: wire `test:server`/`lint`/`format:check` into CI and switch its test step to `npm test` so component tests run.**
- **ESLint + Prettier + EditorConfig (ADR-059)**: mechanical lint/format gate (flat config, TS + Svelte). Landed **config-first**, then a single repo-wide `prettier --write` as a SEPARATE, behavior-preserving commit. Rules start permissive (backlog as `warn`, 0 `error`) so the gate lands without a mass source edit; promote `warn`â†’`error` as the backlog burns down. **Rule: don't hand-fight the formatter; run `npm run format`.**
- **Isolated server test-DB harness (ADR-060)**: `tests/server/**` (run via `npm run test:server`, its own `vitest.server.config.ts`) characterizes the service layer against a throwaway SQLite DB. The `db` singleton reads `$env/dynamic/private`, which Vite **bakes at config-load time** â€” so a `process.env` override inside a setupFile is too late (`db` stays on the dev DB). Retarget via a **shell-env `DATABASE_URL`** set before vitest starts, and gate every run behind a **sentinel-row tripwire** that aborts before any mutation if `db` isn't provably the isolated file. **Rule: a row-count==0 check is NOT a safe tripwire (an empty dev DB false-passes); use a planted sentinel row, and never run a test DELETE until it's found.**
- **Toolchain kept current (ADR-061)**: dev/build/test majors bumped together (Vite 8, vite-plugin-svelte 7, Vitest 4, TypeScript 6, jsdom 29), one major per commit for revertability. All devDeps (zero prod exposure); `svelte`/`@sveltejs/kit` bumped on a separate cadence.

## Interaction sounds (ADR-063)

- **Interaction sounds via vendored `cuelume` (ADR-063)**: subtle Web Audio cues, synthesized live (no audio files). The library (MIT, zero-dep, ~16 kB unminified) is **vendored verbatim** at `src/lib/vendor/cuelume/` rather than npm-installed: at adoption time every published version was younger than the local `min-release-age=7` npm supply-chain policy, and the tarball was source-audited instead (pure Web Audio + delegated listeners; no network/eval/storage; no install scripts). Switch to the npm dep once it ages, if desired. App-owned wrapper `src/lib/sound.svelte.ts` (`initSound`/`setSoundEnabled`/`playSound`); preference = `sound` cookie (default **OFF**), read by the root layout load, toggled from the topbar Appearance menu + /settings â†’ Theming (enabling confirms with a `toggle` cue). **Sound map is deliberately minimal** â€” task completion â†’ `success` (StatusSelect pick, board drag-to-done, bulk status) â€” no hover/press/toggle noise, no declarative `bind()`. Notification chime deferred until notifications get realtime push (the bell only fetches on mount/open â€” no genuine arrival moment). The engine no-ops under SSR, when disabled, and when the browser blocks audio.

## Project sidebar sub-menu visibility (ADR-064)

- **Per-project sidebar sub-menu visibility (ADR-064)**: the sidebar's per-project sub-menu (Overviewâ€¦Settings) shows only a configured subset â€” default **Tasks + Milestones** â€” toggled from a trailing **"â€¦" popover** (checkbox rows). The active route's item always renders even when hidden, so deep links stay oriented; hidden routes stay reachable by URL (visibility â‰  access).
